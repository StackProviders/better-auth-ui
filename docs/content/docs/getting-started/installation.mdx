Follow the steps below to install the `better-auth-ui` library.

<Steps>
<Step>
## Install the Package

Install the `better-auth-ui` library using your preferred package manager.

```package-install
better-auth-ui@latest
```

</Step>

<Step>
## TailwindCSS Configuration

For [TailwindCSS](https://tailwindcss.com) `v4`, add the following `@import` to your global css:

```css title="styles/globals.css"
@import "better-auth-ui/css";
```

- Ensure you have [TailwindCSS](https://tailwindcss.com) `v4` set up and configured correctly.
- You must have [better-auth](https://better-auth.com) `v1.3` or greater installed and configured.
- To use organization components, the **Organization Plugin** must be enabled in your `better-auth` configuration.
  For [TailwindCSS](https://tailwindcss.com) `v3` _(Deprecated)_, add the following `content` to your Tailwind config:

```ts
content: ["./node_modules/better-auth-ui/dist/**/*.{js,ts,jsx,tsx,mdx}"];
```

</Step>

## Manual Installation (Shadcn Registry)

Alternatively, you can install the components directly into your project using the [Shadcn Registry](https://ui.shadcn.com/docs/registry). This method allows you to have full control over the code and customize it as needed.

To install all authentication components:

```bash
npx shadcn@latest add https://stackproviders.github.io/better-auth-ui/r/auth.json
```

To install only the **User Button** component:

```bash
npx shadcn@latest add https://stackproviders.github.io/better-auth-ui/r/user-button.json
```

### Why use Manual Installation?

- **Full Customizability**: The code is copied directly into your `components/auth` folder.
- **No Dependencies**: You don't need to add `better-auth-ui` as a dependency in your `package.json`.
- **Theming**: It automatically uses your existing shadcn UI components (button, input, etc.) and your tailwind theme.
